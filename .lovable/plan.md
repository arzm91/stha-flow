## Objetivo

Remover completamente o editor de planilha (Handsontable/xlsx) e substituir por uma galeria moderna de **8 relatórios pré-definidos** renderizados em HTML e exportados em PDF, com auto-geração via dialog modal em pontos-chave do sistema.

## Os 8 relatórios

| # | Relatório | Escopo | Onde aparece | Auto-geração |
|---|-----------|--------|--------------|--------------|
| 1 | **Estoque Total** | Movimentações, estoque atual, projetado, indicadores | /relatorios + /estoque | — |
| 2 | **Produtividade Total** | Todos equipamentos, última produção, total do mês, estoque atual | /relatorios | — |
| 3 | **Produtividade por Equipamento** | Última produção destacada, matérias-primas consumidas, tabela do mês | /relatorios (seleciona equip) + auto no fim de produção | Dialog ao finalizar produção |
| 4 | **Mensal Consolidado** | Todas produções do mês, mini-dashboard por equipamento (seleção) | /relatorios | — |
| 5 | **Manutenção 24h + Programadas** | Últimas 24h, programadas, indicadores | /relatorios | — |
| 6 | **Ordem de Serviço (Manutenção)** | Dados completos da OS para imprimir | /manutencao + /relatorios (seleciona OS) | Dialog ao criar/programar OS |
| 7 | **Ordem de Produção** | Dados completos da ordem (planejada ou em andamento) | /producao + /relatorios (seleciona ordem) | Dialog ao criar/planejar ordem |
| 8 | **Alertas 24h** | Disparos, causas, efeitos, indicadores | /relatorios | — |

## Renderização de PDF

- HTML/CSS + `window.print()` com `@media print` como caminho primário (rápido, fiel, sem dependências).
- Botão "Baixar PDF" usa `html2pdf.js` ou impressão nativa como PDF.
- Layout A4 responsivo: `max-width: 210mm` em impressão, 100% em tela.
- Header com logo STHA, título, período, data de geração. Rodapé com paginação.

## Auto-geração (dialog modal)

Componente `<AutoGenerateDialog>` reutilizável, aciona-se em:
- **Fim de produção** (mudança de status para "concluída" em `producao.$id`): pergunta "Gerar relatório da produção?" → abre viewer do relatório #3 pré-filtrado.
- **Criação/programação de OS manutenção** (em `manutencao`): oferece OS de manutenção (#6).
- **Criação/planejamento de ordem produção** (em `producao.nova`): oferece ordem de produção (#7).

Preferência dismissível em sessionStorage por tipo ("não perguntar mais nesta sessão").

## Entradas em outras páginas

- `/estoque`: botão "Relatório de estoque" no header abre viewer #1.
- `/manutencao`: botão "Relatório de manutenção" abre #5 + item de menu em cada OS para #6.
- `/producao`: botão "Relatório de produtividade" (#2) + item por ordem para #7.

## Design

- Cada relatório tem header colorido próprio (verde=estoque, azul=produção, laranja=manutenção, vermelho=alertas).
- KPIs em cards no topo com número grande + label + tendência.
- Tabelas com zebra striping, cabeçalho sticky.
- Mini-charts SVG puros (sem lib) para não pesar: barras horizontais para top-N, sparklines.
- Tipografia clara: `text-3xl font-bold` para KPIs, `text-sm` para tabelas.
- Print-friendly: cores mais suaves em `@media print`, quebras de página inteligentes (`break-inside: avoid` em cards e seções).

- Server functions em `src/lib/reports-v2/data/*.functions.ts` com `.middleware([requireSupabaseAuth])`, retornando JSON estruturado (KPIs + tabelas). Nenhum consumo em loader; todos via `useQuery` no viewer.

- Formatação numérica via `format.ts` existente.
- Nenhuma alteração de schema Supabase nesta fase.
